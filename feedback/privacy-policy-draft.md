# Privacy Policy Draft for AllNew App Factory Feedback

最終更新日: 2026-05-19

この文書はプライバシーポリシーへ組み込むためのドラフトです。公開前に日本法の個人情報保護法、Apple Developer Program、App Store Review Guidelines、Sign in with Apple の要件に照らした弁護士レビューを行ってください。

本番公開時点で、プライバシーポリシー、問い合わせ窓口、開示・訂正・利用停止・削除請求、アカウント削除、Apple ID 連携解除の導線を、登録前、投稿前、アカウント設定、公開ページのフッターから常時到達できる状態にします。

本サービスは日本国内在住者向けに提供します。EU、英国、米国カリフォルニア州、中国その他日本国外の居住者を対象にサービス提供する場合、GDPR、UK GDPR、CCPA/CPRA、PIPL その他適用される国外法令への対応を別途確認し、必要な表示、同意、権利行使手続、国外移転条項、アクセス制限または提供地域制限を実装するまで公開しません。

## 1. 取得する情報

当社は、本サービスの提供に必要な範囲で以下の情報を取得します。

| Category | Examples | Public Display |
|---|---|---|
| Apple ID 連携情報 | Apple が提供するユーザー識別子、氏名、メールアドレス、非公開メールアドレス、identity token、authorization code、refresh token 等の認証関連情報 | 公開しない |
| アカウント情報 | 内部ユーザー ID、登録日時、アカウント状態 | 公開しない |
| 投稿情報 | 改善要望、新規アプリ案、タイトル、本文、対象アプリ、公開設定、公開用要約 | 公開される場合がある |
| 公開ニックネーム | ルーレット機能で生成した公開表示名、言語設定、生成ルールのバージョン、使用済み予約状態 | 公開投稿に表示される場合がある |
| 同意ログ | 規約バージョン、プライバシーポリシーバージョン、同意日時、IP アドレス、User-Agent | 公開しない |
| 運用情報 | 審査結果、禁止カテゴリ判定、通報、ブロック、削除処理履歴 | 原則公開しない |

## 2. 利用目的

取得した情報は以下の目的で利用します。

- Apple ID 連携による本人識別、アカウント管理、ログイン維持、Apple ID 連携解除、アカウント削除、token revocation、server-to-server notification への対応
- 改善要望および新規アプリ案の受付、審査、公開、要約、統合、非公開化、削除
- 投稿内容の検討、開発、実装、商品化、結果報告、運営上の分析
- 採用時特典を提供する場合の対象者確認、連絡、App Store Connect で発行および管理される公式プロモコードまたはオファーコードの提供、不正利用防止
- 禁止投稿、権利侵害、スパム、不正利用、セキュリティ事故の防止
- 利用規約、プライバシーポリシー、投稿時同意の証跡管理
- アカウント削除、同意撤回、保有個人データの開示、訂正、追加、削除、利用停止、消去、第三者提供停止、削除請求への対応
- 法令、ガイドライン、Apple Developer Program、App Store Review Guidelines への対応

## 3. 投稿内容の公開

投稿内容は、公開用に要約、編集、統合、匿名化したうえで公開される場合があります。ユーザーは、投稿本文に氏名、住所、電話番号、メールアドレス、健康情報、要配慮個人情報、第三者の秘密情報、営業秘密、未公開発明、特許、実用新案、意匠、商標の出願を予定しているアイデア、第三者の権利を侵害する情報を含めてはなりません。

ユーザーは、登録時および投稿時に、投稿を非公開で受け付けるか、公開ボードへの掲載を希望するかを選択できます。非公開投稿は公開ボードに表示しません。公開ボードへの掲載を希望する場合、当社が提供するルーレット機能で生成された公開ニックネームを表示します。公開ニックネームは自由入力できず、日本語版または英語版を選択できます。当社は、使用済み公開ニックネームを次のユーザーに再提示しないため、公開ニックネーム、言語、生成ルールのバージョン、使用済み予約状態を管理します。公開ニックネームは Apple ID の氏名、メールアドレス、ユーザー識別子とは分離して管理します。ただし、公開ニックネームは匿名性を保証するものではなく、投稿内容、時期、文脈その他の事情により投稿者が推測される可能性があります。

当社は公開前審査と個人情報検出を行いますが、投稿者自身が記載した情報の完全な検出または削除を保証しません。

公開前審査キュー、禁止カテゴリ判定、内部審査理由、削除判断、通報内容、権利侵害またはプライバシー侵害に関する具体的な判断ログは、公開面には表示しません。公開する場合は、運営担当者が確認した公開用要約、ステータス、状況報告、結果報告、統計情報または運用方針に限定します。AIその他の自動処理を用いる場合であっても、公開可否、削除、要約、権利侵害またはプライバシー侵害に関する最終判断は人間の運営担当者が行います。

## 4. 第三者提供および外部送信

本番実装では、以下を公開画面またはプライバシーポリシー内で具体的に明示します。

- Sign in with Apple の利用、Apple 提供ユーザー識別子、メールアドレス、非公開メールアドレス、token revocation、server-to-server notification の取扱い
- ホスティング、データベース、ログ管理、監視、メール送信等の委託先
- 送信される情報の項目
- 送信目的
- 委託先または第三者が講じる保護措置

広告目的の第三者提供またはトラッキングは行わない設計を原則とします。将来分析ツール等を追加する場合は、外部送信規律、Apple のプライバシー要件、同意撤回導線を再確認します。

### 4A. 外国にある第三者への提供

Sign in with Apple の利用に伴い、Apple Inc.（米国）に対して、Apple 提供ユーザー識別子、メールアドレス、非公開メールアドレス、identity token、authorization code、refresh token 等の認証関連情報が送信される場合があります。当社は、本番公開時点で、個人情報保護法第28条に基づき、外国にある第三者への個人データ提供を認める旨の本人同意、提供先の国名、当該外国の個人情報保護制度に関する情報、提供先が講じる個人情報保護措置、本人の求めに応じた情報提供方法を、登録前および投稿前に明示します。

外国制度に関する情報は、個人情報保護委員会が公表する外国制度情報および提供先事業者のプライバシー公表情報を参照し、ユーザーが容易に確認できる形でリンクまたは要約を掲示します。米国以外の外国に所在するホスティング、データベース、ログ管理、監視、メール送信、問い合わせ管理、セキュリティ対策等の委託先を利用する場合は、提供先名、所在国、送信項目、利用目的、保護措置を本番公開前に追加します。日本国内リージョンのみで処理する委託先についても、国外再委託または国外からのアクセス可能性を確認し、必要な場合は外国第三者提供または外部送信として明示します。

### 4B. Cookie、外部送信、トラッキング

本サービスは、ログイン維持、不正利用防止、アカウント削除、通報・削除申出、開示等請求対応に必要な必須CookieおよびセッションCookieを使用します。広告SDK、広告目的の分析SDK、行動ターゲティング、第三者広告トラッキングSDKは使用しない設計を原則とします。

本番公開時には、改正電気通信事業法の外部送信規律への対応として、外部送信先、送信される情報、送信目的、送信先での利用目的、オプトアウト可否、Cookieの有効期間、SameSite属性、Secure属性、HttpOnly属性を、Cookie/外部送信ポリシーまたは本ポリシー内で通知または公表します。

## 5. 保管期間

| Data | Retention |
|---|---|
| アカウント情報 | アカウント削除完了まで |
| 投稿本文 | 公開・運営上必要な期間。削除請求時は個人データ部分を削除または匿名化 |
| 同意ログ | 規約バージョン、同意文言、投稿ID、同意日時、権利譲渡および対価請求不可の証跡は、最終利用またはアカウント削除から10年。ただし法的紛争中は必要期間 |
| IP アドレス / User-Agent | 生ログは原則6か月。その後は削除、匿名加工情報の作成基準を満たす形での匿名化、または仮名加工情報に相当する不可逆ハッシュ化。ただし不正調査、法令対応、紛争対応に必要な場合は必要期間 |
| セキュリティログ | 原則1年。ただし不正調査、法令対応、紛争対応に必要な場合は必要期間 |
| 通報・審査ログ | 原則3年。ただし再発防止、紛争対応に必要な場合は必要期間 |

## 6. アカウント削除、同意撤回、開示等

ユーザーは、アカウント設定または当社所定の問い合わせ先から、アカウント削除、Apple ID 連携解除、同意撤回、保有個人データの開示、訂正、追加、削除、利用停止、消去、第三者提供停止を請求できます。この窓口は本番公開時点で稼働させ、本人確認方法、受付時間、回答方法、手数料の有無、対応できない場合の理由を明示します。

Sign in with Apple を利用するアカウントを削除する場合、本番では Apple REST API による token revocation を実施します。ユーザーが Apple 側で連携解除または Apple Account 削除を行い、当社が server-to-server notification その他の通知を受領した場合も、アカウント無効化、Apple ID 連携解除、メール到達性の確認、投稿公開範囲の確認を行います。

同意撤回、Apple ID 連携解除、アカウント削除、個人データ削除が行われた場合でも、撤回または削除前に投稿済みの内容について既に成立した権利譲渡、対価請求不可、著作者人格権不行使、商品化許諾、秘密情報ではないことの確認、証跡保存に関する合意は、法令上許される範囲で維持されます。削除後も、法令上保存が必要な情報、権利譲渡、対価請求不可、紛争対応に必要な同意ログは、目的、期間、アクセス権限を限定して保存する場合があります。

## 6A. 健康情報および要配慮個人情報

本サービスは改善要望と新規アプリ案の受付を目的とし、診断、治療、予防、医療助言、健康データ管理、緊急対応を目的としません。体温、血糖、体重、服薬、症状、診断名、検査結果、その他個人の健康情報または要配慮個人情報は投稿しないでください。

健康関連アプリに関する投稿で健康情報が混入した場合、当社は公開前に削除、匿名化、非公開化、投稿者への確認、または削除請求対応を行います。本体アプリが扱う健康データの取扱いは、各アプリのプライバシーポリシーおよびアプリ内説明に従い、本サービスの投稿データとは分離して管理します。

受付できるのは「測定履歴を週単位で比較したい」「入力メモ候補を増やしたい」のような表示、記録、操作性に関する改善要望です。「血糖値からインスリン量を提案してほしい」「症状から治療方針を判定してほしい」のような診断、治療、投薬判断、医療助言は受付できません。

## 7. 安全管理措置

本番では以下を実装します。

- 管理者認証と最小権限
- 投稿、同意ログ、審査ログの改ざん防止
- TLS による通信保護
- 保存データの暗号化
- 個人情報と公開投稿データの分離
- 監査ログ、異常検知、インシデント対応手順
- HSTS、CSP、SameSite Cookie、Secure Cookie、HttpOnly Cookie、CSRF対策、Originチェック、レート制限、通報スパム対策
- server-to-server notification 受信用エンドポイントの署名検証、リプレイ防止、監査ログ保存

## 8. 未成年

本サービスは18歳未満の利用を認めません。Apple ID連携のみでは年齢確認にならないため、登録時および投稿時に「私は18歳以上です」と確認するチェックボックスを設けます。Sign in with Apple の `realUserStatus` 等を利用できる場合でも、それは不正利用対策の補助情報として扱い、年齢確認の代替とはしません。18歳未満の利用が判明した場合、当社はアカウント停止、投稿の非公開化、削除その他必要な措置を行います。

本ポリシーおよび本サービスに関する紛争は、日本法を準拠法とし、法令上許される範囲で、東京地方裁判所または東京簡易裁判所を第一審の専属的合意管轄裁判所とします。

当社は、本ポリシーを変更する場合があります。重要な変更を行う場合は、公開ページ、登録画面、投稿画面、メールその他当社が適切と判断する方法により、変更内容、効力発生日、ユーザーが取り得る手続を事前または速やかに通知します。
