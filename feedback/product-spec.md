# AllNew App Factory Feedback Portal Product Spec

## 1. Purpose

AllNew App Factory で開発・公開したアプリに対する改善要望と、新しいアプリのアイデアを受け付ける公開ポータルを提供する。

利用料・登録料は無料。ユーザー登録は Apple ID 連携のみとし、投稿内容は運営会社が無償で自由に利用できる前提を、登録時と投稿時に明示する。権利譲渡には、著作権法第27条および第28条に規定する権利、特許を受ける権利、実用新案登録を受ける権利、意匠登録を受ける権利を含める。採用時特典を提供する場合でも、当該特典は任意であり、投稿または権利譲渡の対価ではない。採用時特典は App Store Connect で発行および管理される公式のプロモコード、オファーコードその他 Apple が認める公式コードに限定し、独自の利用クレジット、独自コード、残高管理、ポイント台帳は提供しない。本サービスは日本国内在住者向けとし、国外居住者向けに提供する場合はGDPR、UK GDPR、CCPA/CPRA、PIPL等の多法域対応を完了する。

公開名称では Apple との提携・後援・承認を示唆しない。Apple という語は Apple ID 連携、App Store、Apple の公式ガイドライン参照など、正確な説明が必要な場面に限定する。

## 2. User Roles

| Role | Capability |
|---|---|
| Guest | 公開済み要望、ステータス、結果報告を閲覧できる |
| Registered user | Apple ID 連携後に改善要望・新規アプリ案を投稿できる |
| Reviewer | 投稿を審査し、公開可否、統合、却下、禁止カテゴリ判定を行う |
| Operator | ステータス、状況報告、結果報告、公開範囲を管理する |
| Privacy operator | アカウント削除、同意撤回、開示・削除請求、Apple ID token revocation を処理する |

## 3. Public Statuses

| Status | Meaning | Public Commitment |
|---|---|---|
| 受け付けました | 投稿を受領し、公開可能な形式で受付済み | 検討・対応を約束しない |
| 検討しています | 採否、実現性、優先度を確認中 | 実装を約束しない |
| ごめんなさい | 対応を見送った状態 | 理由は公開できる範囲で表示 |
| 対応しています | 実装、検証、審査準備等を進めている状態 | 完成・公開・継続提供を約束しない |
| 出来ました | 一定の対応を実施した状態 | 投稿内容どおりの完全実装を保証しない |

## 4. Core Workflow

1. Guest が公開ボードを閲覧する。
2. 投稿時は Apple ID で登録またはログインする。
3. 登録時に利用規約、プライバシーポリシー、投稿ガイドラインへ同意し、投稿の初期公開設定を `非公開` または `公開希望` から選択する。初期値はプライバシー保護のため `非公開` とする。
4. `公開希望` を選ぶ場合、公開ニックネームをルーレットで生成する。ニックネームは自由入力させず、何回でも再生成できる。日本語版と英語版を用意し、英語版は日本語版の軽いノリを引き継ぎつつ、英語圏の文化、宗教、礼儀に配慮して、宗教的称号、政治的語、攻撃的語、差別的語、過度に身分的または威圧的な語を避ける。
5. 生成後は、禁止用語辞書に含まれる語を照合し、該当する場合は再抽選する。使用済みの公開ニックネームはサーバー側で予約し、次のユーザーの候補として再提示しない。公開名はニックネームのみとし、Apple ID の氏名、メールアドレス、ユーザー識別子は公開しない。
6. 投稿直前に、18歳以上かつ日本国内在住者であること、無償利用、権利譲渡、著作権法第27条および第28条の特掲、対価請求不可、採用時特典が任意かつ非対価であること、結果非保証、禁止カテゴリ非該当、特許出願予定アイデア・未公開発明・営業秘密の不投稿をチェックする。
7. 投稿ごとに `非公開で受け付ける` または `公開ボードに出す` を選択できる。非公開投稿は運営のみが確認し、公開ボード、公開統計、公開ステータス更新の対象にしない。
8. 投稿は非公開の審査キューへ入る。審査中の投稿ID、本文、禁止カテゴリ判定、詳細な審査理由は公開しない。
9. Reviewer が禁止カテゴリ、個人情報、健康情報、第三者秘密情報、営業秘密、未公開発明、第三者権利、重複、公開可否、Apple/プラットフォーム規約違反、通報対象の有無を確認する。
10. 公開可能な投稿だけを、運営担当者が確認し、個人情報・秘密情報・権利侵害情報を除去または要約したうえで、公開ニックネームとともに `受け付けました` として公開する。AIは補助に限り、公開可否、削除、要約、権利侵害またはプライバシー侵害に関する最終判断は人間が行う。
11. Operator が状況報告と結果報告を更新する。
12. 対応を中止・延期・変更・見送りする場合はステータスと結果報告を更新する。
13. ユーザーが削除請求または同意撤回を行った場合、投稿の公開継続可否を審査し、個人データ削除、Apple ID 連携解除、トークン失効を処理する。

## 6. Prohibited Categories

以下は受付不可とする。

- 反社会的勢力、反社会的勢力の利益となるもの
- 犯罪、違法行為、脱法、詐欺、薬物、武器、不正アクセス等を助長するもの
- たばこ、電子たばこ、喫煙関連商品、武器、弾薬、違法薬物、非合法に処方された規制薬物、偽造品、盗品、消費者安全上の高リスク商品
- 宗教活動、布教、寄付勧誘、信仰の勧誘を主目的とするもの
- 政治活動、選挙、投票誘導、政治的主張の拡散を主目的とするもの
- 出会い、恋愛、婚活、性的マッチング、面会斡旋を主目的とするもの
- アダルト、性的表現、風俗、性的サービス、ポルノに関するもの
- 差別、ヘイト、誹謗中傷、嫌がらせ、暴力、不寛容、自傷を助長するもの
- 個人情報、プライバシー、肖像、名誉、信用を侵害するもの
- 著作権、商標権、特許権、営業秘密その他第三者の権利を侵害するもの
- 特許、実用新案、意匠、商標の出願を予定しているアイデア、未公開発明、営業秘密、第三者秘密情報
- 医療、金融、法律、投資、ギャンブル等の高リスク規制領域
- Apple、App Store、その他プラットフォーム規約に違反するおそれがあるもの
- Apple または第三者との提携、後援、承認、公式性を誤認させるもの

## 7. Moderation and User Safety

App Store Review Guideline 1.2 の UGC 基準に合わせ、本番では以下を必須とする。

- 公開前の人手審査
- 禁止カテゴリ、個人情報、健康情報、第三者秘密情報、営業秘密、未公開発明、権利侵害、スパムの自動検出
- 公開投稿の問題報告ボタン
- 不適切ユーザーのブロック、アカウント停止、再投稿制限
- 運営への問い合わせ先と削除請求導線
- 24時間以内を目安とした緊急通報レビュー
- 監査ログの改ざん防止と担当者操作履歴
- 公開面には、公開前審査中、非公開化済み、公開済み要約、通報・削除申出の受付方針だけを表示する
- 公開用要約、削除、編集、統合、非公開化は運営担当者が確認して実行する。AIその他の自動処理は補助に限り、人間の承認なしに公開判断を確定しない
- プロバイダ責任制限法、情報流通プラットフォーム対処法の適用可能性を、投稿規模と公開範囲に応じて定期確認する

## 8. Privacy and Account Deletion

本番では以下を必須とする。

- プライバシーポリシーの常時表示
- 利用目的、取得項目、第三者提供、外部送信、保管期間、削除請求、同意撤回、開示、訂正、利用停止、第三者提供停止を明示
- アカウント設定から削除開始可能にする
- Apple ID 連携利用者の削除時に Apple REST API で token revocation を実施する
- Apple の server_to_server_notification を受信し、署名検証、リプレイ防止、アカウント無効化、Apple ID連携解除、監査ログ保存を行う
- Apple Inc.（米国）への認証関連情報送信について、個人情報保護法第28条に基づく外国第三者提供同意、国名、外国制度情報、提供先の保護措置、本人求めへの情報提供方法を明示する
- Cookie/外部送信ポリシーとして、必須Cookie、セッションCookie、外部送信先、送信項目、送信目的、送信先利用目的、オプトアウト可否、SameSite/Secure/HttpOnly属性、有効期間を明示する
- 投稿本文に個人情報が含まれる場合の非公開化と削除・匿名化フロー
- 法令上保存が必要な同意ログは、保存理由と期間を記録して分離保管する
- 同意ログ、規約バージョン、同意文言、投稿ID、権利譲渡および対価請求不可の証跡は10年を上限に保存する。生のIPアドレスとUser-Agentは原則6か月で削除、匿名化、または不可逆ハッシュ化する
- 同意撤回またはアカウント削除後も、投稿済み内容について成立済みの権利譲渡、対価請求不可、著作者人格権不行使、商品化許諾は法令上許される範囲で維持されることを登録前・投稿前に明示する
- 匿名化は匿名加工情報の作成基準を満たす形で行い、不可逆ハッシュ化は仮名加工情報相当として個人情報性が残り得る前提で管理する

## 8A. Age and Region Eligibility

- 本サービスは18歳以上かつ日本国内在住者のみ利用可能とする
- Apple ID連携のみでは年齢確認にならないため、登録時および投稿時のチェックボックスで18歳以上であることを確認する
- Sign in with Apple の `realUserStatus` 等を利用できる場合も、不正利用対策の補助情報として扱い、年齢確認の代替とはしない
- EU、英国、米国カリフォルニア州、中国その他日本国外の居住者を対象に提供する場合は、GDPR、UK GDPR、CCPA/CPRA、PIPL等の対応を完了してからローンチする

## 8B. Adoption Benefit Policy

- 当社は、改善要望または新規アプリ案を採用または参考にして当社のアプリまたはサービスを公開した場合、任意で採用時特典を提供できる
- 採用時特典は、App Store Connect で発行および管理される公式のプロモコード、オファーコードその他 Apple が認める公式コードに限定する
- 独自の利用クレジット、独自コード、残高管理、ポイント台帳その他 App Store Connect 外の特典管理機能は持たない
- 特典の内容、有効期間、対象範囲、利用条件、提供時期、提供方法、提供可否は、App Store Connect の仕様、上限、利用条件、対象アプリまたはアプリ内課金の状態、および当社の裁量に従う
- 採用時特典は、投稿、アイデア、発明、著作物、ノウハウ、権利譲渡、採用、実装、公開、販売、収益化の対価、報酬、ロイヤリティ、成功報酬、補償または損害賠償ではない
- 投稿者に共同開発者、著作者、発明者、権利者、販売代理人、収益分配権者その他これらに類する地位を認めない
- 特典は App Store Connect または App Store 上で利用できる範囲に限られ、現金交換、払戻し、譲渡、換金、差額請求、代替特典の請求はできない
- 不正投稿、禁止カテゴリ該当、第三者権利侵害、虚偽申告、複数アカウント、規約違反が判明した場合は、特典を提供しないことができる

## 9. Sign in with Apple Production Requirements

- 本番では Apple 提供の Sign in with Apple JS を使用する
- ボタンは Apple Human Interface Guidelines に準拠し、独自色・独自ロゴ・紛らわしい表現にしない
- Sign in with Apple はユーザーが任意にアカウント作成・サインインする目的に限定する
- Apple ID 連携以外のログインを追加する場合、App Store Review Guideline 4.8 の同等ログイン要件を再確認する
- アカウント作成があるため、Webおよび該当アプリ内からアカウント削除を開始できる導線を提供する
- アカウント削除時はApple REST APIの token revocation を実行し、Apple側の連携解除またはApple Account削除に伴う server_to_server_notification を処理する

## 10. Business Operator and Legal Display

- 本番公開ページに、AllNew合同会社の事業者名、代表者、所在地、連絡先、問い合わせ受付方法、受付時間を表示する
- 無料サービスであっても、特定商取引法、消費者契約法、個人情報保護法、Apple Developer Program、App Store Review Guidelines の表示・連絡先要件を公開前に確認する
- 利用規約、プライバシーポリシー、投稿ガイドライン、削除申出、開示等請求、アカウント削除の導線を登録前・投稿前・フッター・アカウント設定に置く

## 11. Health Data Boundary

- 本サービスは改善要望・新規アプリ案の受付用であり、診断、治療、予防、医療助言、健康データ管理、緊急対応を目的としない
- 体温、血糖、体重、服薬、症状、診断名、検査結果、その他個人の健康情報または要配慮個人情報は投稿禁止にする
- ThermoSnap、GlucoSnap、WeightSnap 等の健康関連アプリに関する要望で健康情報が混入した場合は、公開前に削除、匿名化、非公開化、または投稿者への確認を行う
- 本体アプリの健康データ取扱いと、フィードバックポータルの投稿データを分離し、各アプリのプライバシーポリシーと整合させる
- 受付できる例: 「測定履歴を週単位で比較したい」「入力メモ候補を増やしたい」のような表示、記録、操作性の改善
- 受付できない例: 「血糖値からインスリン量を提案してほしい」「症状から治療方針を判定してほしい」のような診断、治療、投薬判断、医療助言
