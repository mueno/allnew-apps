# AllNew App Factory Feedback Portal

AllNew App Factory で開発・リリースしたアプリの改善要望と新規アプリ案を受け付け、審査し、対応状況と結果を公開するための静的 MVP です。

## Artifacts

- `index.html` - ローカルで開ける静的ポータル画面
- `app.js` - 投稿、認証デモ、ステータス表示のクライアントスクリプト
- `product-spec.md` - サービス仕様、運用フロー、データ項目
- `ux-concept-2026.md` - サービスコンセプト、理想ユーザー動線、画面設計原則
- `legal-draft.md` - 利用規約に入れるべき法務条項ドラフト
- `privacy-policy-draft.md` - プライバシーポリシードラフト
- `deployment-security.md` - 本番配信時のセキュリティヘッダーと運用要件
- `audit-report.md` - 9回の監査・是正ループの結果

## Local Preview

`docs/feedback-portal/index.html` をブラウザで開くと確認できます。静的 HTML/CSS/JS のため、開発サーバーは不要です。

## UX Direction

Apple Support アプリを参考に、一般ユーザーがスマホで迷わず使える情報設計へ寄せています。画面設計の正本は `ux-concept-2026.md` です。

- ホームは「対象アプリ」「サポートツール」「よく見る項目」に絞り、投稿、公開ボード、ステータス、ルールを別画面として段階的に開く
- 常時表示のサイドバー／ボトムメニューは使わず、右上のハンバーガーメニューに主要導線を集約する
- 文字サイズは iOS の標準スケールに合わせ、Large Title 34px、行タイトル 17px、補足 13px を基準にする
- 44px 以上の操作ターゲット、横スクロール可能な対象アプリカード、丸いグループリストを使い、スマホ優先で設計する
- 受付不可カテゴリ、権利譲渡、結果非保証など重い説明は投稿直前とルール画面に分けて表示する
- 投稿画面は「何を送るか」「どう扱うか」「同意して送る」の3フェーズに分け、公開範囲、ニックネーム、受付不可カテゴリ、非保証を送信前に確認できるようにする
- 公開ボードはステータスをカード上部に固定し、運営が公開できる状況・結果報告だけを読みやすく表示する。該当なしの場合は空状態を出し、非公開投稿数や内部審査理由は表示しない
- ルール画面は要点カードと開閉式の詳細で構成し、長文の法務説明を必要なときだけ読めるようにする
- ワクワク感は、Apple HIG の控えめな色使い、明確な階層、意味のあるモーションを基準に、AllNew ロゴのブルーから青〜シアンのトーナル配色へ統一する。強い補色ではなく、光沢、深度、押下反応、小さな浮遊バッジで楽しく見せる
- ヘッダーには AllNew コーポレートロゴを反映し、サポートツールの主要アイコンは生成画像をリサイズして軽量化したグラデーションアイコンを使う
- AllNew アプリの紹介は Apple Store アプリのカード型ディスカバリーに寄せる。ただし、投稿受付の利用条件として他アプリのダウンロードや購入を要求せず、任意の発見導線として扱う
- 公開/非公開は登録時の初期値と投稿ごとの選択に分ける。非公開を初期値にし、公開希望時だけ自由入力不可のルーレット式ニックネームを何回でも回せる遊びとして置く。日本語版と英語版を用意し、生成後は不適切語の禁止用語辞書と使用済みニックネーム台帳に照合する

## Scope

この MVP は以下を確認するための実装です。

- Apple ID 連携のみで登録する前提の画面導線
- 改善要望と新規アプリ案の投稿フォーム
- 登録時と投稿時の公開/非公開選択、公開時の日本語/英語ルーレット生成ニックネーム、使用済みニックネームを次の候補から外す挙動、非公開投稿を公開ボードに表示しない挙動
- 投稿前の権利譲渡・無償利用・対価請求不可・結果非保証の明示同意
- 採用時特典を提供する場合の任意性、非対価性、App Store Connect 公式コード限定、独自管理機能なし、現金化・譲渡・払戻し不可の明示
- 著作権法第27条および第28条の特掲、特許を受ける権利、実用新案登録を受ける権利、意匠登録を受ける権利の明示譲渡
- アカウント削除、Apple ID 連携解除、同意撤回、開示・削除請求の本番導線
- 公開ステータス: `受け付けました` / `検討しています` / `ごめんなさい` / `対応しています` / `出来ました`
- 反社、犯罪、宗教、政治、出会い系、アダルト等の受付不可カテゴリ表示
- 公開前審査の方針、公開状況、結果報告の画面骨格
- 運営側の審査キュー、禁止カテゴリ判定、詳細な審査理由、削除判断ログは内部管理に限定し、公開面には表示しない設計
- 健康関連アプリの機能要望と医療助言の境界FAQ
- 監査ログ保存期間、公開用要約の人間確認、ローンチ前ゲート
- Apple Inc.（米国）への外国第三者提供、Cookie/外部送信、国内在住者限定、多法域対応方針、18歳確認、強行法規優先、準拠法・管轄の方針

## Production Notes

この静的公開版は閲覧専用として公開します。ローカルプレビューではUI検証のため疑似ログインを有効にしますが、`apps.allnew.work` などの公開ホストでは疑似ログインと疑似投稿を無効化し、Apple ID認証、サーバー側検証、投稿保存、公開前審査キューが接続されるまで投稿受付を開始しません。

本番化では、Apple Sign in、ユーザー ID、投稿 ID、規約バージョン、同意日時、公開設定、公開ニックネーム、ニックネーム言語、ニックネーム生成ルールバージョン、使用済みニックネーム予約日時、外国第三者提供同意バージョン、Cookie/外部送信表示バージョン、監査ログ保存期間ポリシー、採用時特典の文言バージョンをサーバー側で保存してください。生のIPアドレスとUser-Agentは原則6か月で削除、匿名加工情報の作成基準を満たす匿名化、または仮名加工情報に相当する不可逆ハッシュ化を行い、同意ログと権利譲渡証跡は10年を上限に保存する設計です。投稿本文に含まれる個人情報・健康情報・第三者秘密情報・営業秘密・未公開発明の検出、管理者審査、公開前の非公開キュー、通報、ブロック、削除請求導線も必須です。Apple token revocation、server-to-server notification、外国第三者提供同意、Cookie/外部送信表示、アカウント削除、開示等請求窓口、事業者情報表示、採用時特典をApp Store Connect公式コードに限定する運用、本番セキュリティヘッダーを実装し、公開前には弁護士レビューを行ってください。
